Description

gdk-pixbuf 库提供了一个简单的机制用来加载和显示动画。一个动画从概念上讲就是一系列随 时间显示的帧。所有的帧都一样大小。动画内部可能并不是表现为一些列的帧；比如，它可能储存 一个小精灵(sprite)和使之绕背景移动的指令。要显示一个动画您并不去要理解它的表现形式， 而只要询问 gdk-pixbuf 在给定的时间点需要显示什么。

gdk_pixbuf_animation_get_iter ()

GdkPixbufAnimationIter * gdk_pixbuf_animation_get_iter  (GdkPixbufAnimation *animation,
                                                         const GTimeVal *start_time);

得到一个显示动画用的迭代子(iterator)。该迭代子提供了在给定的时间显示的帧 (frame)。可以使用 g_object_unref() 将其释放

start_time 通常可以由 g_get_current_time() 得到，且标记该动画回放的开始。生 成迭代子后，应该立即显示由 gdk_pixbuf_animation_iter_get_pixbuf() 返回的 pixbuf。这时，应该设定一个计时器(timeout)(使用 g_timeout_add())或者通过其他机 制确保你可以在由 gdk_pixbuf_animation_iter_get_delay_time() 得到的数值的毫秒 之后更新图像，你需要重新设定定时器到新的，有可能改变的延迟时间

作为一个捷径，如果 start_time 设置为 NULL, 将自动使用 g_get_current_time() 的结果(result)

要更新图片(例如，要将 gdk_pixbuf_animation_iter_get_pixbuf() 的结果变成动画的 新的帧)，调用 gdk_pixbuf_animation_iter_advance()。

如果正在使用 GdkPixbufLoader，除了在延迟时间之后更新图像外，还需要在收到 area_updated 信号且gdk_pixbuf_animation_iter_on_currently_loading_frame() 返 回 TRUE 时更新图像。在这种情况下，当前放在这个 loader 里的帧接收到了新的数 据，所以需要刷新。该帧的延迟时间也应该在收到 area_updated 信号后被修改，比如 一个帧的延迟时间被编码在该帧本身之后的数据里。所以定时器应该在收到任何 area_updated 信号之后被修改。

延迟时间有可能为 -1，表示"无限长."

gdk_pixbuf_animation_iter_advance ()

gboolean            gdk_pixbuf_animation_iter_advance   (GdkPixbufAnimationIter *iter,
                                                         const GTimeVal *current_time);

可能更新一个动画到新的帧。这个帧的选择依赖于传递给 gdk_pixbuf_animation_get_iter() 的开始时间

current_time 通常由 g_get_current_time() 得到，而且必须大于或等于传递给 gdk_pixbuf_animation_get_iter() 的时间，而且当每次 gdk_pixbuf_animation_iter_get_pixbuf() 被调用时必须增加或保持不变。

作为一个捷径，为当前时间传递 NULL 则 g_get_current_time() 将被自动介入。如果 你想做一些象加倍速度播放的怪事，只需要为 current_time 传递确切的值。

如果该函数返回 FALSE，将不需要更新动画显示，假定先前渲染的显示为更新; 如果 是 TRUE，则需要调用gdk_animation_iter_get_pixbuf() 且用新的 pixbuf 更新显 示。

iter :

 一个 GdkPixbufAnimationIter

current_time :

 现在的时间

Returns :

 如果图像需要更新则返回 TRUE

gdk_pixbuf_animation_iter_get_pixbuf ()

GdkPixbuf *         gdk_pixbuf_animation_iter_get_pixbuf
                                                        (GdkPixbufAnimationIter *iter);

得到应该在当前显示的 pixbuf；这个 pixbuf 应该和这个动画一样大小 (gdk_pixbuf_animation_iter_get_width()，gdk_pixbuf_animation_get_height())。 这个 pixbuf 应该被显示 gdk_pixbuf_animation_iter_get_get_delay_time() 个毫秒 长。这个函数的调用者并不拥有返回的 pixbuf 的引用，任何时候调用 gdk_pixbuf_animation_iter_advance() ，这个迭代子（iterator）更新到下一个帧时 这个 pixbuf 便失效了。如果想再更新了迭代子后重复利用该 pixbuf，拷贝这个 pixbuf (不要只增加一个引用)。

 一个动画的迭代子（iterator）

 将显示的pixbuf
