gdk-pixbuf-csource

gdk-pixbuf-csource — 为 GdkPixbuf 图像生成 C 代码的工具。

Synopsis

gdk-pixbuf-csource [选项] [图像]

gdk-pixbuf-csource [选项] --build-list [ [名称] [图像] ...]

描述

gdk-pixbuf-csource 是一个可以生成包含图像的 C 代码的小工具 , 可以用来将图像直接编译进程序。

祈祷(Invocation)

gdk-pixbuf-csource 带一个作为输入的单个图像文件名，或者，使用 --build-list 选项， 一个由 (name, image) 对 组成的列表来为指定为 name 参数的一列图像生成代码。

选项

--stream

生成的 pixbuf 数据流 (一个按网络字节序(network byte order)包含被序列化(serialized)的 GdkPixdata 结构的字符串)。

--struct

生成 GdkPixdata 结构(需要 gdk-pixdata.h 文件的 GdkPixdata 结构定义)。

--macros

为这个图像生成 *_ROWSTRIDE, *_WIDTH, *_HEIGHT, *_BYTES_PER_PIXEL 和 *_RLE_PIXEL_DATA 或 *_PIXEL_DATA 宏定义。

--rle

为生成 pixel 数据启用 run-length 编码(默认)。

--raw

为生成 pixel 数据禁用 run-length 编码。

--extern

生成 extern 符。

--static

生成 static 符(默认)。

--decoder

提供一个 *_RUN_LENGTH_DECODE(image_buf, rle_data, size, bpp) 宏定义来解码 run-length 编码。

--name=identifier

为生成的变量或宏(只当未指定 --build-list 有效)指定 identifier name(前缀)。

--build-list

启用 (name, image) 对解析模式。

-h, --help

打印简要帮助并退出。

-v, --version

打印版本号并退出。

--g-fatal-warnings

将警告(warning)设置成致命错误(使程序终止)。

参见

GdkPixbuf 文档，随 Gtk+ 一同发布，在 www.gtk.org 可以找到。

错误(Bugs)

The runlength encoder gets out of sync with the pixel boundaries, since it includes the rowstride padding in the encoded stream. Furthermore, it generates pixbufs with suboptimal rowstride in some cases.[这句还真不会翻译]
