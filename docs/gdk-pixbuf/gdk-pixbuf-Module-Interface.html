Description

如果 gdk-pixbuf 被编译为支持 GModule，它就能通过可以加载（和解析、保存）新的图像和动画 格式的模块来进行扩展。每个可加载的模块都必须有一个名叫fill_info 的 GdkPixbufModuleFillInfoFunc 的出口函数和一个名叫 fill_vtable 的 GdkPixbufModuleFillVtableFunc 出口函数。

为了在实际加载模块前进行格式检查(format-checking)工作，模块用过 fill_info 输出他们的特性（和其他信息）。gdk-pixbuf-query-loaders 是一个外部 工具，它可以用来创建一个包含所有可用的加载器(loader)列表和他们的特性的文本文件。这个文件当 gdk-pixbuf 在运行时读取来收集可用的加载器的列表和他们的特性。

模块可能只实现 GdkPixbufModule 功能的一个子集。如果一个特定的功能没有实现， fill_vtable 函数将不会设置 GdkPixbufModule 结构的相应函数指针。 如果一个模块支持渐进加载(例如:提供 begin_load, stop_load 和 load_increment)，将 需要再实现 load，因为 gdk-pixbuf 可以提供一个通用加载的实现来包装这个渐进加载。

安装一个模块需要两步。

  • 将模块的文件拷贝到加载器(loader)目录(一般为： libdir/gtk-2.0/version/loaders， 除非被环境变量 GDK_PIXBUF_MODULEDIR 重写(override)了。)

  • 调用 gdk-pixbuf-query-loaders 来更新模块文件 (一般为： sysconfdir/gtk-2.0/gdk-pixbuf.loaders， 除非被环境变量 GDK_PIXBUF_MODULE_FILE 重写了。)

gdk-pixbuf 借口需要实现的模块包含在 gdk-pixbuf-io.h ( 如果这个模块支持动画，还包含在 gdk-pixbuf-animation.h 中)中。 gdk-pixbuf 正常API的稳定性保证并不包含它们。为了强调这个事实，它们被 #ifdef GDK_PIXBUF_ENABLE_BACKEND 所保护。

struct GdkPixbufModulePattern

struct GdkPixbufModulePattern {
	char *prefix;
	char *mask;
	int relevance;
};

模块的签名(signature)是一个前缀集(a set of prefixes)。前缀被编码成原始字符串对 (pairs of ordinary strings)。如果第二个字符串(叫做掩码 mask)非空 (not NULL )， 则必须与第一个字符串(前缀)一样长，而且可以包含' '(空格), '!', 'x', 'z', 和 'n' 分别表示该字节 必须匹配，不匹配， 不关心("don't-care"-bytes)， 零 和 非零。 每个前缀都有一个相关的正数用来描述与前缀的相关性(relevance)，0表示错配(mismatch) ，100表示完全匹配(perfect match)。

从 gdk-pixbuf 2.8开始，掩码的第一个字节可以是 '*'，表示不只从开头开始 匹配，也可以从中间开始，叫做unanchored pattern。2.8以前的版本会认为 '*' 是一个 'x'。

模块的签名被储存为一个 GdkPixbufModulePattern 的数组。该数组以一个 prefix 为 NULL 的模式结尾。 

GdkPixbufModulePattern *signature[] = {
  { "abcdx", " !x z", 100 },
  { "bla", NULL,  90 },
  { NULL, NULL, 0 }
};
在这个例子中，比如 "auud\0" 匹配的相关性为(relevance)100，而 "blau" 匹配的 相关性为90。

GdkPixbufModuleSizeFunc ()

void                (*GdkPixbufModuleSizeFunc)          (gint *width,
                                                         gint *height,
                                                         gpointer user_data);

定义一个函数类型，这个函数将在所加载的图像的大小被确定的时候被调用。

这个函数将用来设置 width 和 height 到图像期望被缩放的大小。如果一个模块没有 一个高效的方法在图像加载过程中实现预期的缩放，将要么忽略这个(设置)大小的请求，要么 只是估算它 -- gdk-pixbuf 将在图像完全加载之后再进行预期的缩放。

如果这个函数将 width 或 height 设置成 0，这个模块将其理解成它将很快被关闭而且不需要 分配更多的资源。这个约定用来更高效的实现 gdk_pixbuf_get_file_info()。

struct GdkPixbufModule {
	char *module_name;
	char *module_path;
	GModule *module;
	GdkPixbufFormat *info;

                                     GError **error);

                                     const guchar *buf,

GdkPixbufModule 包含了以特定的格式加载和保存图像所需要的函数。

GdkPixbufModule 可以被 GModule 动态加载。每个可加载的模块必须包含一个名为 fill_vtable 的 GdkPixbufModuleFillVtableFunc 函数， 这个函数将在这个模块被加载时调用而且必须设置 GdkPixbufModule 的函数指针。
